# Документация проекта аналитики планировок

## Описание проекта
Это веб-приложение для анализа планировок с использованием искусственного интеллекта. Приложение позволяет загружать PDF-файлы с планировками, анализировать их и получать детальные отчеты.

## Технологический стек
Проект построен на современных веб-технологиях:
*   **Фреймворк**: [Next.js 16](https://nextjs.org/) (App Router)
*   **UI Библиотека**: [React 19](https://react.dev/)
*   **Стилизация**: [TailwindCSS 4](https://tailwindcss.com/)
*   **Работа с PDF**: `pdf-lib`, `react-pdf`
*   **AI**: Google Generative AI (@google/generative-ai)

## Установка и запуск

### Предварительные требования
Для работы с проектом вам понадобится [Node.js](https://nodejs.org/) (рекомендуется версия 20 или выше).

### Установка зависимостей
1.  Клонируйте репозиторий (если еще не сделали этого):
    ```bash
    git clone <ссылка-на-ваш-репозиторий>
    cd <папка-проекта>
    ```

2.  Установите зависимости, используя `npm`, `yarn`, `pnpm` или `bun`:
    ```bash
    npm install
    # или
    yarn install
    # или
    pnpm install
    # или
    bun install
    ```

3.  Настройте переменные окружения:
    Создайте файл `.env.local` в корне проекта (если требуется) и добавьте необходимые API ключи (например, для Google Generative AI).

### Запуск в режиме разработки
Для запуска локального сервера разработки выполните команду:

```bash
npm run dev
# или
yarn dev
# или
pnpm dev
# или
bun dev
```

Откройте [http://localhost:3000](http://localhost:3000) в вашем браузере.

## Сборка и тестирование

### Сборка проекта (Production)
Для создания оптимизированной сборки приложения:

```bash
npm run build
```

После успешной сборки вы можете запустить приложение в продакшен-режиме:

```bash
npm run start
```

### Линтинг и проверка кода
Для проверки кода на наличие ошибок и соответствие стилю используйте:

```bash
npm run lint
```

## Работа с Git / GitHub

Ниже приведены основные команды для работы с системой контроля версий:

### Статус и добавление изменений
Проверить статус измененных файлов:
```bash
git status
```

Добавить все изменения в индекс (подготовить к коммиту):
```bash
git add .
```

### Сохранение изменений (Commit)
Зафиксировать изменения с комментарием:
```bash
git commit -m "Ваше описание изменений"
```

### Отправка изменений (Push)
Отправить изменения в удаленный репозиторий (например, на GitHub):
```bash
git push origin main
# или другая ветка, например master/develop
```

### Получение изменений (Pull)
Получить и слить изменения из удаленного репозитория:
```bash
git pull origin main
```
